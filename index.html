WORK ABOUT

PUBLIC HEALTH ONTARIO REDESIGN

A REDESIGN OF PUBLIC HEALTH ONTARIO FOR A MORE EFFICIENT AND PLEASANT USER EXPERIENCE

Tools:

Optimal Sort, Figma, Google Docs

Timeline:

3 weeks

My Role:

Everything from heuristic analysis, user research, visual design to prototyping

THE PROBLEM:

Through user interviews and testing we found that users had trouble navigating the website to find the information they needed. Users were overwhelmed by the amount of information on the website.

THE SOLUTION:

The information architecture of the website was reorganized to create a simpler navigation. The homepage of the website was redesigned to have only the most crucial information and it was laid out in a more accessible way. Overall the redundancies in navigation were fixed for a less confusing experience.

RESEARCH

I investigated the website to gain more insight on their main purpose. Based on this I created a proto persona to illustrate an example of a possible user. The main audience of the wesite seemed to be researchers and meical professionals based on the heavy emphasison scientific reports and medical data.

User interviews were done to investigate the usability of the current website. 5 users were asked to find the cancer page on the website and then to open an article on that page. They were the asked about their overall experience with the website. In addition to the interviews, we did a heuristic analysis and an analysis of the colour scheme for accessibility to pin point how the website was divided and which aspects were crucial to usability.

I organized the information gained from this process into a prioritiation matrix to come up with the main issues with the current website.

KEY ISSUES:

• Users found navigating home from a page confusing

• One of the colour combinations did not pass accessibility

• The home page was overwhelming for users

• A lot of the links such as the covid related information links and the login button went to the same pages and thus were redundant

• Users were confused about the difference between “Health topics and diseases and conditions”

• All of the information in items in the “Data and Analyses” tab is also in other pages not under that tab

IDEATION AND FIRST PROTOTYPE

To begin my redesign I reorganized the information into a site map. I used an open card sort task to help me in this. I only included the items in the primary and secondary navigation in this task as not to overwhelm the participants. 6 participants participated in the card sort and they categorized the items into 4 categories on average.

In the first prototype I focused on making the navigation and home page less confusing and overwhelming based on the issues discovered in user interviews. I categorized the items on the homepage and reduced the amount of reports presented. The idea was that based on grouping the user would be able to find the information they needed in a shorter amount of time. I also got rid of some of the redundancies in navigation to make the path to a specific page more clear and reduce the amount of categories in the navigation. For example, the ‘Data and Analysis’ tab included the same categories as the other items in the primary navigation. For this reason I eliminated this category and decided to include a “Data and Analysis” section on the actual topic pages.

VISUAL DESIGN

To update the visuals of the website I generated a style guide for the new design. I aimed to keep some of the branding from the first website but make it cleaner and more approachable. Users did not find the existing colour scheme unappealing so I kept it pretty similar for the redesign. The purple, however, seemed to clash with the rest of the colours so I omitted it, simplifying the colour scheme. The concept of the style was to make the site feel professional and scientific. I also made a new logo for the site to make it look more professional as well as being more modern. In this step I created the grids for the desktop version as well as for the mobile to make the responsive design consistent. I went with a 12 column design for the desktop and a 4 column design for the mobile for flexibility in laying out the different types of elements on both platforms.

HIGH FIDELITY PROTOTYPES AND TESTING

While applying the visual design I wanted complement the simplified organization. I tried to use a lot of white space and minimized the amount of colours used. I also went through a few ideas for the home page before landing on the final one.

This design was tested by 10 users who were asked to go through a series of tasks to test both the desktop and the mobile prototype. They first needed to find the injuries page, then the poison page. They then needed to be able to return to the injuries page and then back home. In the second task they needed to find the COVID-19 data tool and favourite the page.

During these tests users had the following issues:

• There was some misspelling in the breadcrumbs

• Some users felt that injuries did not fall under diseases and conditions and only chose that tab by default

• Users felt that hover states in the tabs and in the favourite buttons were missing

• Some users were not sure where they would find their favourited items

• Some users were not clear on what the stars meant or though hearts would be better suited

• Users were not sure how to get home on the mobile version

• Some users preferred to search instead of using the navigation

Bearing these comments in mind I made the following changes:

• Added hover states for the star buttons and the nav dropdowns

• Tool tips were added to indicate that the stars add the items to read later

• Added a button at the top to show the amount of items added to read later

• Added home button to mobile version

FINAL PROTOTYPE

SUMMARY

The way the information on the Public Health Ontario website was organized was overwhelming and confusing for users. Through this redesign I focused on reorganizing the website to make the navigation easier for the user. I simplified the colour scheme, redesigned the logo and paired down the navigation. Following the user testing I added a new feature in the form of “Read Later” tab at the top of the page to indicate where to find favourited items. In the next steps the website can be further improved by applying the UI to more pages. Additionally, there could be more microinteractions added to make the experience of using the website more delightful.

WHAT I LEARNED

In this project I not only got to consider the user but also the organization. I was challenged by the amount of information that needed to be on the website and how to organize it all in a way that would be accessible to the user. I implemented the card sort task which helped a lot in doinf that. This is something I plan to keep doing in future projects. I also found that users often tried to use the search button instead of the primary navigation to try to find an article. I feel that for the fututre it would be good to consider this and implement changes to the search button which may make it even more user friendly. Additionally this was my first time coming up with an entire style guide and I made the mistake of not saving the pieces of it as components right away to use in my prototype. This made my prototyping process less efficient than I would have liked. I am now aware of the importance of saving pieces of the design as components to make quick iterations.
